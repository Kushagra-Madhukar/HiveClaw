# Brand Assets

This folder contains reusable HiveClaw branding assets derived from the current favicon and repository identity.

## Assets

- [HiveClawFaviconMonoLight.png](./HiveClawFaviconMonoLight.png)
  - white-on-transparent favicon variant
  - use on dark backgrounds, terminal docs, and dark UI surfaces

- [HiveClawFaviconMonoDark.png](./HiveClawFaviconMonoDark.png)
  - charcoal-on-transparent favicon variant
  - use on light backgrounds and print-like surfaces

- [HiveClawSocialPreview.svg](./HiveClawSocialPreview.svg)
  - 1280x640 social/share preview composition
  - suitable as a base asset for GitHub social cards, docs hero previews, and release imagery

## Source Assets

- [HiveClawFavicon.png](../HiveClawFavicon.png)
- [HiveClawTransparent.png](../HiveClawTransparent.png)
- [HiveClawSolid.png](../HiveClawSolid.png)

## Usage Guidance

- Prefer `HiveClawFavicon.png` for repository and docs branding when you want the full-color mark.
- Prefer `HiveClawFaviconMonoLight.png` or `HiveClawFaviconMonoDark.png` where the mark needs to sit cleanly on constrained backgrounds.
- Keep the favicon variants as small-surface assets. For future website/app branding, a dedicated vector redraw is still the correct long-term move.
